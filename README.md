🎯 Objetivo do Software

Criar um sistema desktop eficiente, capaz de realizar o gerenciamento de patrimônios da Universidade de Brasília, mesmo em condições instáveis de internet.
O software deve:

Funcionar completamente offline, utilizando rede local.
Garantir segurança e organização das informações.
Automatizar o registro, consulta e controle de patrimônios.
Atender às necessidades dos funcionários do Laboratório Latitude e professores do Departamento de Engenharia Elétrica.

Este problema foi identificado a partir da experiência prática do desenvolvedor dentro da própria universidade.

📋 Funcionalidades do Sistema (Requisitos)
✅ Requisitos Funcionais
[RF001] Listagem de Patrimônios disponíveis diariamente
Ator: Funcionários do Laboratório Latitude
Descrição: Exibe todos os patrimônios disponíveis para empréstimo.
[RF002] Cadastro de Patrimônios
Ator: Funcionários
Descrição: Permite cadastrar novos patrimônios, incluindo dados e condição de uso.
[RF004] Relatórios
Ator: Funcionários
Descrição: Geração de relatórios para auxiliar professores no processo de empréstimo.
⚙️ Requisitos Não Funcionais
[RNF001] Sistema Desktop

O software deve ser totalmente desktop devido à instabilidade de internet da Universidade de Brasília, permitindo funcionamento offline usando apenas a rede local.

[RNF002] Linguagem de Programação

O sistema deve ser desenvolvido em Java, garantindo compatibilidade com ambientes desktop e cumprindo os requisitos do projeto.
